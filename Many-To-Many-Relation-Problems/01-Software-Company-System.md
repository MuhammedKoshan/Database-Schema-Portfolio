## 📝 Problem Statement: Software Company System

Design a relational database schema for a software company based on the following business rules:

1. **Departments:** The company is divided into multiple departments (e.g., AI, Web, Embedded Systems).
2. **Programmers:** The company employs several programmers. Each programmer must belong to exactly one department, while a single department can have many programmers.
3. **Projects:** The company manages various software projects.
4. **Assignments:** A programmer can work on multiple projects simultaneously. Similarly, each project requires a team of multiple programmers.
5. **Tracking:** The database must track the specific number of **hours** each programmer spends working on each individual project.

**Objective:** 
Create an Entity-Relationship Diagram (ERD) that accurately maps these entities, resolves any Many-to-Many (M:N) relationships, and correctly places all Primary Keys (PK), Foreign Keys (FK), and relationship attributes.
